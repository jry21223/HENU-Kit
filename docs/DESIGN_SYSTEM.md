# HENU-Kit Design System

> 状态：Baseline v1.0（2026-07-16）  
> 适用范围：HENU-Kit 网站、学习资料库、刷题产品及相关开源项目的用户界面  
> 本文描述 HENU-Kit 自有视觉系统，不代表河南大学官方视觉识别规范。

本文是所有 HENU-Kit 学生端产品的设计基线。新增页面和功能必须遵守；确需偏离时，应先在 Issue 或 Pull Request 中说明原因、影响范围和替代方案，再修改实现。

## 1. 品牌定位

HENU-Kit 是面向河南大学学生的校园工具与信息入口，由学生自主发起、运营和维护。

### 必须出现的非官方声明

正式介绍、网站页脚、README 和对外宣传中，应明确使用以下表述：

> HENU-Kit 是由河南大学学生自主发起并维护的校园工具项目，非河南大学官方产品，不代表学校官方立场。

页脚等空间有限的场景可使用短版：

> 学生自主运营 · 非河南大学官方项目

### 禁止使用的表述

- “河南大学官方工具箱”
- “河大官方平台”
- “学校指定平台”
- 未经证实的“河南大学官方标准色”或“河大官方设计”

## 2. 品牌原则

### 2.1 学生视角

用学生熟悉的任务语言组织产品，例如“找资料”“开始刷题”“今天吃什么”，避免行政系统式命名。

### 2.2 清楚直接

页面首先说明用户能完成什么，不用宏大口号掩盖操作入口。

### 2.3 有校园感，但不冒充官方

可以从校园生活、纸张、档案、书本和学生共建中获得灵感，但不得复制河南大学校徽或暗示官方授权。

### 2.4 一个系统，多种工具

不同模块可以拥有不同内容密度，但必须共享颜色、排版、导航、按钮、卡片和状态反馈规范。

### 2.5 功能克制

- 首页只承担发现、进入和继续上次任务，不复制各业务站的完整功能。
- 一个页面应围绕一个主要任务组织；不能因为“以后可能需要”提前堆入口。
- 新功能必须回答：解决谁的什么问题、为什么不能由现有模块承担、如何验证有效。
- 优先链接和复用已有能力，避免重复建设第二套资料库或第二套刷题引擎。

## 3. 色彩系统

当前配色来自 HENU-Kit Demo 自身的视觉设计，不是河南大学官方 VI。以下名称均为 HENU-Kit 内部品牌命名。

### 3.1 主色：Kit 墨绿

**Kit 墨绿（Kit Ink Green）** 是 HENU-Kit 的主品牌色。

| Token | 色值 | 用途 |
|---|---:|---|
| `--hk-ink-green` | `#0C6B45` | 主按钮、品牌标志、重点标题、选中状态 |
| `--hk-ink-green-deep` | `#05603A` | Hover、按下状态、深色强调区域 |
| `--hk-ink-green-soft` | `#E4EFE9` | 浅色标签、成功提示、选中背景 |

“墨绿”强调的是纸张与油墨的编辑感；“Kit”明确它属于 HENU-Kit，而不是学校官方标准色。

### 3.2 基础色

| Token | 色值 | 用途 |
|---|---:|---|
| `--hk-paper` | `#F5F1E7` | 页面主背景，称为“纸白” |
| `--hk-paper-raised` | `#FFFFFF` | 卡片、弹层、输入区域 |
| `--hk-paper-line` | `#E9E6DB` | 分割线、边框、禁用背景 |
| `--hk-ink` | `#343D36` | 正文与主要文字 |
| `--hk-ink-muted` | `#6F766F` | 次要说明与元信息 |

### 3.3 强调色：Kit 麦金

| Token | 色值 | 用途 |
|---|---:|---|
| `--hk-wheat-gold` | `#F0BE44` | 插画、徽标、少量重点提示 |
| `--hk-wheat-gold-soft` | `#F8E8B4` | 提示背景、轻量装饰 |

麦金只用于点亮信息层级，不作为大段正文颜色，也不与白色文字组合成主要按钮。

### 3.4 色彩使用比例

- 纸白与白色：约 70%
- 墨绿与文字色：约 20%
- 麦金及其他强调色：不超过 10%

每个页面只允许一个主要强调色。美食、学习、校园工具不应各自发展一套互不相关的主题色。

## 4. 排版

### 4.1 字体建议

```css
font-family: Inter, "Noto Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif;
```

不依赖字体图片呈现品牌名称。Logo 字标和页面标题必须在不同设备上保持可读。

### 4.2 字级

| 层级 | 桌面端 | 移动端 | 用途 |
|---|---:|---:|---|
| Display | 64–80 px | 40–48 px | 首页核心标题，仅一处 |
| H1 | 40–48 px | 32–36 px | 页面标题 |
| H2 | 28–32 px | 24–28 px | 区块标题 |
| H3 | 20–24 px | 18–20 px | 卡片和内容标题 |
| Body | 16–18 px | 16 px | 正文 |
| Meta | 13–14 px | 13–14 px | 标签、日期、辅助信息 |

大标题可以有编辑式张力，但操作页面应降低字号，优先保证扫描效率。

## 5. 间距与形状

- 基础间距单位：`4px`
- 常用间距：`8 / 12 / 16 / 24 / 32 / 48 / 64px`
- 卡片圆角：`12px`
- 大型展示卡片圆角：`20px`
- 胶囊按钮只用于短标签和少量主要动作，不应让所有控件都变成胶囊
- 阴影保持克制，优先使用边框、留白和背景层级

## 6. 核心组件

### 6.1 全局产品外壳

所有学生端产品共享：

- HENU-Kit 品牌入口
- 一致的主导航结构
- 一致的登录/个人入口位置
- 移动端底部导航规范
- 统一的非官方声明

模块名称作为 HENU-Kit 下的功能名称出现，例如“HENU-Kit · 学习”，不再建立彼此竞争的独立品牌。

### 6.2 按钮

- Primary：Kit 墨绿背景，白色文字
- Secondary：透明或纸白背景，墨色文字，浅色边框
- Tertiary：纯文字按钮，用于低优先级操作
- Danger：仅用于删除、退出等不可逆或高风险操作

同一视区内原则上只出现一个 Primary 动作。

### 6.3 卡片

卡片必须承载清晰任务，不做纯装饰容器。课程卡片、美食卡片和校园工具卡片共享边框、圆角和文字层级，仅内容结构不同。

### 6.4 状态

必须统一设计：

- Loading
- Empty
- Error
- Success
- Disabled
- Login required

错误信息使用人能理解的语言，并提供明确的下一步。

### 6.5 表单与输入

- 每个输入框必须有可见标签，placeholder 不能代替标签。
- 必填、格式和错误提示应在字段附近说明，不只用颜色表达。
- 提交后保留用户已输入的非敏感内容；失败时不得要求全部重填。
- 验证码输入支持粘贴和自动聚焦，但不得阻止用户手动编辑。

### 6.6 导航

- 桌面端一级导航不超过 5 项；移动端底部导航不超过 5 项。
- 一级入口按学生任务命名：`首页 / 找资料 / 去刷题 / 校园生活 / 我的`。
- “美食榜单”属于校园生活；“资料库”和“刷题”保持独立任务入口。
- 外部或独立站点跳转必须保持统一页头或给出明确的产品归属提示。

## 7. 产品结构与模块边界

HENU-Kit 是统一入口与产品外壳，不是将所有项目代码合并为一个单体应用。

| 模块 | 唯一职责 | 不承担 |
|---|---|---|
| HENU-Kit 主站 | 产品入口、搜索/发现、账户入口、状态与公告 | 复制资料内容、实现第二套题库 |
| 资料库 `study.superhuazai.me` | 浏览、搜索、预览和下载学习资料 | 刷题与测验流程 |
| 刷题站 `superhuazai.me/practice` | 练习、进度、错题和结果反馈 | 资料归档与综合门户 |
| 校园生活 | 美食榜单等轻量校园信息 | 与学习工具混合成复杂工作台 |

模块可以独立部署，但必须复用设计 token、全局导航、账户状态、页脚声明和基础组件。

## 8. 响应式与可访问性

- 以移动端为首要体验，关键流程必须在 360px 宽度下可用。
- 正文行宽建议不超过 72 个中文字符；长文档提供清晰目录。
- 可点击区域最小 `44 × 44px`；正文不小于 `16px`。
- 正文和背景对比度至少达到 WCAG AA；焦点状态不可移除。
- 所有图片必须提供有意义的 `alt`，纯装饰图使用空 `alt`。
- 动画应短而克制，并尊重 `prefers-reduced-motion`。
- 键盘可以完成登录、搜索、筛选和提交等核心流程。

## 9. 文案规范

- 使用简体中文和学生日常表达，句子短、动作明确。
- 按钮使用动词：`查看资料`、`开始刷题`、`发送验证码`，避免模糊的“确定”。
- 不使用“官方”“权威”“指定”等可能造成主体误认的词。
- 不承诺无法保证的结果，例如“100% 抢到”“绝对准确”。
- 错误文案包含发生了什么和下一步，例如“验证码已过期，请重新获取”。
- 空状态优先给出可执行动作，不用段子替代说明。

## 10. 统一账户与隐私界面

- 统一账户入口使用 `account.henukit.cn`（实施前完成域名配置）。
- 学生邮箱验证码用于证明邮箱控制权；界面不得把它描述成学校官方认证或校方授权。
- 发件域名使用独立子域名，例如 `notify.henukit.cn`，不得使用暗示官方身份的发件人名称。
- 验证码有效期建议 5–10 分钟、单次使用；重发需等待至少 60 秒并设置邮箱/IP 频率限制。
- 密码、验证码、Token 和 Cookie 不得出现在页面日志、错误提示、截图和公开仓库中。
- 收集字段遵循最小化原则；注册前说明用途，提供退出登录和删除账户/数据申请入口。
- 登录状态、失效状态和跨站跳转反馈必须保持一致。

## 11. Logo 规范与当前判断

### 11.1 GitHub 仓库现有 Logo

当前 `assets/logo.jpg` 为黑白字标：

- 主文字：`HENU Kit`
- 副文字：`对不起 我是外包`

它有明显的学生社区幽默感，适合作为：

- GitHub README 彩蛋
- 社区海报
- 限定版头像或贴纸
- 开发者文化素材

不建议直接作为正式产品主 Logo，原因是：

1. 没有可独立使用的图形标志，缩小后无法用于 favicon、App 图标和头像。
2. 副文字依赖大尺寸，移动端和小图标场景不可读。
3. “我是外包”与“学生自主运营”的正式定位存在语义冲突。
4. JPG 白底版本难以适配深色背景、透明背景和多尺寸输出。

因此将其定义为：**Community Edition / 社区趣味版本**，保留但不作为正式主标。

### 11.2 当前网站“河”字图标

单独使用“河”字容易让部分用户联想到河南大学官方标志。考虑到项目必须明确非官方，该图标不应在未经进一步评估的情况下成为最终正式 Logo。

### 11.3 正式 Logo 的最低要求

正式 Logo 尚未定稿。后续方案必须满足：

- 有可独立使用的图形符号
- 24px 下仍可识别
- 不复制或改造河南大学校徽
- 不使用容易造成官方误认的印章式构图
- 能与 `HENU Kit` 字标组合，也能单独作为 favicon
- 提供 SVG、透明 PNG、单色和反白版本

优先探索 `HK` 字母组合、模块/连接结构或工具箱抽象符号，不优先使用单独的“河”字。

## 12. 非官方身份的展示位置

| 场景 | 要求 |
|---|---|
| 网站首页 | 首屏说明或首屏后第一个可信信息区出现完整声明 |
| 全站页脚 | 固定出现“学生自主运营 · 非河南大学官方项目” |
| 登录与支付 | 在涉及账号、资料权限或付费前再次明确主体 |
| README | 项目简介后直接出现完整声明 |
| 海报与二维码 | 使用短版声明，不得仅依靠链接页解释 |

非官方声明应清晰可见，但不使用警告红或夸张弹窗制造不必要的风险感。

## 13. 设计交付与验收

页面进入开发前至少具备：

- 目标用户、主任务和成功条件
- 移动端与桌面端关键页面
- Loading / Empty / Error / Success / Disabled 状态
- 组件与 token 对照，不在业务页私造颜色和间距
- 登录、未登录和权限不足状态
- 文案、数据来源和更新时间说明

合并前检查：

- 主流程可在手机上独立完成
- 同一视区只有一个 Primary 动作
- 非官方声明位置正确
- 键盘焦点、对比度和触控尺寸合格
- 不重复资料库或刷题站的既有能力

## 14. 规范治理

- 本文件采用语义版本：破坏性规则调整升主版本，新增兼容规则升次版本，文字澄清升修订版本。
- 颜色、Logo、全局导航、账户和模块边界属于核心决策，修改必须经过团队评审。
- 新组件先证明至少有两个真实复用场景，再进入共享组件库。
- 业务页面临时样式应设清理期限，不得无限期成为事实标准。
- 每次版本发布在文末记录日期、决策和负责人。

## 15. 已确定与待决策事项

当前已经确定：

- 保留现有深绿色视觉方向
- 将主色命名为“Kit 墨绿”，不称“河大绿”
- HENU-Kit 是学生自主运营的非官方项目
- GitHub 现有 Logo 作为社区趣味版本保留
- 正式产品 Logo 仍需单独设计和验证

正式 Logo 确定后，应在本文补充安全区、最小尺寸、组合方式和禁用示例。

仍待团队确认：

- 正式 Logo 方向与交付时间
- `henukit.cn` 及账户、通知子域名最终配置
- 统一登录的会话方案和跨站接入顺序
- 共享组件库采用的具体技术实现
