# HENUKit 按学院分类资料与竞赛目录

快照日期：2026-07-17
最近官网核验与目录修订：2026-07-22

## 本包结构

- `00_校级公共资料`：校级细则、教务处竞赛工作说明、竞赛通知全文；
- `01_人文社科类` 至 `05_国际合作类`：按照河南大学官方院系目录创建的单位文件夹；
- `06_其他研究与培养单位`：本轮取得正文、但不在上述单位中的单位；
- `90_开发文档`：资料更新Hook、数据库、事件Schema和竞赛扩展方案；
- `98_范围审核`：明确不属于本科生范围或正文范围尚未核实的条目；
- `99_总清单与元数据`：学院与竞赛的CSV/JSON总表及文件哈希清单。

## 使用边界

1. 只有官网正文或官方附件明确面向本科生的通知，才进入本科生索引。
2. `list_extracted`、`official_page_verified_summary`、`blocked_*` 等状态不代表完整正文；必须先看条目的 `extraction_status`。
3. `full_structured_extract` 表示正文所有实质章节已按结构提取，内容是核对用的结构化改写，不是逐字复制；同时保留官方原件时，应以原件为最终依据。
4. 历史版规则标明适用届次，不能直接视为2026届现行规则。
5. 名单公示仅保留官方链接和元数据，不复制姓名、学号等个人信息。
6. 研究生奖学金、研究生评先奖优等明确面向研究生的文件已移入 `98_范围审核/明确不纳入本科生范围`；无法确认本科生范围的文件移入 `98_范围审核/待核验范围`。
7. 生命科学学院的 canonical 目录为 `02_自然科学类/生命科学学院`；学院资料不得再建立第二个同名分类目录。
8. 校级竞赛正文以 `00_校级公共资料/02_校级竞赛目录与通知` 为 canonical 资料源；学院目录中的同文通知是离线导航副本，更新时必须以内容哈希校验同步。

## 提取状态约定

- `full_structured_extract` 及其 `full_structured_extract_*` 变体，以及 `full_extracted_with_source_inconsistency`：正文已按章节完成结构化提取；若标注 `historical`、`source_inconsistency` 或 `zip_unavailable`，不得仅凭完整提取状态切换为 current。
- `official_notice_metadata_only`：官方页面可核验，但只保留标题、日期、链接等元数据，典型场景是名单公示或含个人信息的结果页。
- `list_extracted`：仅从官方列表/首页取得标题和日期，不等于正文完整；若已找到详情页，记录 `source_url` 为详情页，并将列表地址保存为 `source_list_url`。
- `blocked_405`、`blocked_or_unavailable`：一次访问快照，不是永久属性；重新访问成功后必须更新状态。
- `superseded_by_full_record`、`historical_superseded`：仅用于历史访问记录或已被新版替代的资料，不进入 current。

## 竞赛边界

- `discipline_relevance` 是HENUKit推荐映射，不是学院官方认定。
- 研究生专属赛事不纳入本科生通用来源索引；只对特定高年级本科生开放的赛事需看当年章程，不应概括为所有本科生均可参加。
