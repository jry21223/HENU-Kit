# HENUKit 迁移与实施顺序

## 阶段一：把现有资料从“链接索引”迁移为“正文版本”

1. 按 `99_总清单与元数据/file_manifest.json` 扫描本包中的 Markdown；完整正文实际分散在 `00_校级公共资料`、`01_人文社科类` 至 `06_其他研究与培养单位`，不存在名为 `完整提取` 的目录；
2. 解析 YAML 元数据和正文，按根 README 中的 extraction status 约定分类；
3. 为每份材料创建 `documents` 和首个 `document_versions`；
4. `blocked_405`、`blocked_or_unavailable`、`list_extracted` 和 `official_notice_metadata_only` 创建候选/元数据记录和人工任务，不设为 current；
5. 生成全文索引，暂缓向量化，先验证结构化字段。

## 阶段二：上线只读采集器

1. 接入教务处、学生处、团委和一个学院；
2. 只生成 diff 和候选事件，不自动切换 current；
3. 连续运行两周，统计误报、页面模板变化和 405 比例；
4. 为每个来源保存回归测试 HTML 样本。

## 阶段三：启用版本关系和内部 Hook

1. 建立 Transactional Outbox；
2. 接入 HMAC 签名和幂等消费；
3. 只对“明确替代声明”和“同政策家族年度继任”自动切换；
4. 其他变化进入人工审核。

## 阶段四：接入知识库

1. 按章节语义分块；
2. 旧版本分块保留但设为非 current；
3. 默认检索只查 current；
4. 历史查询必须显示年份和有效状态；
5. 对权重冲突、附件缺失和正文受限材料禁止生成确定性结论。

## 验收标准

- 同一页面仅访问量变化不触发更新；
- 新附件同名但内容变化可识别；
- 新年度细则上线后旧版本仍可追溯；
- 受限页面不会覆盖完整规则；
- Hook 重放不会重复生成分块；
- current 切换、索引更新和事件发送可审计。
